# CONTRIBUTING

感谢你为 `cailema-skill（当前仓库目录名可保留）` 贡献内容。

本仓库是 **Agent Skill 文档仓库**，不接收前端、后端、数据库、API 等无关代码。

## 先看范围

欢迎贡献：
- 新场景（被裁、HR沟通、背调、转行、副业等）
- 话术模板（可复制、可落地）
- 决策流程优化（decision-tree/workflows）
- 示例对话改进（examples）
- 清单模板完善（resources/checklist）

不接受：
- 网站/服务端功能开发
- 违法、报复、泄密、删数据等内容
- 法律确定性结论或医疗/心理诊断建议

## 如何提 Issue

请优先使用 Issue 模板：
- `Bug Report`：内容错误、链接错误、结构不一致
- `Feature Request`：希望新增能力（不改项目定位）
- `Scenario Request`：新增真实场景
- `Prompt Improvement`：改进提示词/输出结构

提 Issue 时建议写清：
1. 你遇到的问题是什么
2. 发生在什么文件/场景
3. 期望结果是什么
4. 可选：你建议的改法

## 如何提 PR

1. Fork 仓库并创建分支
2. 只修改与本次目标相关的文件
3. 本地自检后提交 PR
4. 使用仓库 PR 模板说明改动

PR 必填信息：
- 改了什么
- 为什么改
- 对哪些场景有帮助
- 是否影响 `examples` / `scripts` / `decision-tree`

## 如何提交新的 Skill 场景

推荐步骤：
1. 先在 `examples/` 新增一个场景对话文件
2. 再补充对应 `resources/` 清单或说明
3. 如有必要，更新 `decision-tree.md` 的分流说明

场景文件建议最少包含：
- `用户输入`
- `Skill 输出`（按统一 8 段结构）

## 如何改进 scripts / examples / workflows

### 改进 scripts.md

- 每条话术要可直接复制发送
- 优先短句，减少模糊表达
- 明确适用对象（HR/领导/家人/面试）

### 改进 examples/

- 使用真实问题而非抽象提问
- 输出尽量遵循统一 8 段结构
- 明确“今天做什么”和“本周做什么”

### 改进 workflows.md

- 保持固定顺序：情绪稳定 -> 局面判断 -> 决策分流 -> 信息与策略 -> 行动清单 -> 风险提醒 -> 时间线 -> 行动收束
- 每个流程写成步骤，不写空泛原则

## 如何保持风格一致

风格基线：
- 冷静、现实、有同理心
- 不鸡汤、不说教、不高高在上
- 多行动建议，少大道理
- 像行业老兵，像深夜聊天，但结构清晰

写作检查清单：
- [ ] 是否能在 30 秒内看懂“下一步做什么”
- [ ] 是否包含可执行动作（今天/本周）
- [ ] 是否包含可复制话术
- [ ] 是否包含风险提醒
- [ ] 是否避免法律确定性表达
