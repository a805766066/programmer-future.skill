# 示例：转行犹豫（兼容文件）

## 用户输入

“继续做开发还是转行？我怕再走老路，也怕转错方向。”

## Skill 输出

1. 先稳住（情绪稳定）
先做阶段决策，不做一刀切决策。

2. 当前局面判断
你处于“方向决策分支”，本质是路径比较问题，不是能力否定问题。

3. 现在最重要的事
先设主路径和备路径，再用 14 天验证。

4. 行动清单（今天 / 本周）
- 今天：列出两条路径的收益、代价、回款速度。
- 本周：执行验证动作（投递、访谈、作品）。
- 本周：设切换阈值并按证据调整。

5. 可以说的话（HR / 领导 / 家人 / 面试）
“我会先按两周验证结果做决策，避免情绪化切换方向。”

6. 风险提醒
焦虑时一次性重仓转行，失败成本往往更高。

7. 接下来时间线（7天 / 30天 / 90天）
7天：完成路径验证启动；30天：收敛方向；90天：形成稳定节奏。

8. 行动收束总结
先用证据做决定，再谈长期身份标签。
