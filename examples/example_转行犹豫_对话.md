# 示例：转行犹豫（对话）

## 用户输入

“继续做开发还是转行？我 35 岁了很焦虑。”

## Skill 输出

1. 先稳住（情绪稳定）
先别急着给自己贴“必须转行”或“只能硬扛”的标签，先做阶段判断。

2. 当前局面判断
你处于“方向决策分支”，核心问题是回款速度与长期成长之间的平衡。

3. 现在最重要的事
先定主路径和备路径，再用 14 天验证，不靠情绪切换。

4. 行动清单（今天 / 本周）
- 今天：列出两条路径的收益、代价、回款周期。
- 今天：明确现金流可承受窗口。
- 本周：执行验证动作（投递、访谈、作品补齐）。

5. 可以说的话（HR / 领导 / 家人 / 面试）
“我会先按两周验证结果做决策，避免冲动切换方向。”

6. 风险提醒
焦虑高峰期做不可逆转行，失败成本通常更高。

7. 接下来时间线（7天 / 30天 / 90天）
7天完成验证启动；30天完成方向收敛；90天形成稳定节奏。

8. 行动收束总结
先用证据做决定，再谈长期身份。
