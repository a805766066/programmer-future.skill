# 示例：是否出海远程（对话）

## 用户输入

“有人建议我做出海远程岗位，我不知道值不值得转过去。”

## Skill 输出

1. 先稳住（情绪稳定）
先别被“远程高薪”标签刺激，先评估真实可执行性。

2. 当前局面判断
你处于“转行分析分支/方向决策分支交叉场景”，核心是语言、时区、协作模式与稳定性。

3. 现在最重要的事
确认你是否具备远程岗位的交付能力和沟通能力。

4. 行动清单（今天 / 本周）
- 今天：核对目标岗位对英语、时区和协作工具要求。
- 今天：准备远程协作案例和可展示作品。
- 本周：投递 5-10 个远程岗位并跟踪反馈。

5. 可以说的话（HR / 领导 / 家人 / 面试）
“我愿意做远程岗位，但会优先选择交付边界清晰、沟通机制成熟、结算稳定的团队。”

6. 风险提醒
只看薪资不看结算规则、合同约束和时区成本，后续压力会很大。

7. 接下来时间线（7天 / 30天 / 90天）
7天完成岗位可行性评估；30天看真实面试反馈；90天决定是否作为主路径。

8. 行动收束总结
先小规模验证，再决定是否全面切换到出海远程。
